本次更新重点补齐了从 `curl` 到请求编辑器的导入链路，让常见的抓包结果和命令行请求可以更快落到工作台里直接复用。现在把整段 `curl` 粘贴到 URL 输入框后，应用会自动解析出请求地址、方法、请求头和 Cookie，减少手动拆解参数的成本。

## curl 粘贴导入增强

- URL 输入框现在支持直接粘贴完整的 `curl` 命令，并自动解析为当前 HTTP 请求。
- 解析结果会同步填充请求方法、URL、查询参数、请求头、认证信息和请求体。
- 新增了对常见 `curl` 参数的识别，包括 `-X`、`-H`、`-u`、`-d`、`--json`、`-F` 和 `--form-string`。
- URL 输入框的占位提示已经更新，会明确提示支持直接粘贴 `curl` 命令。

## Cookie 透传修正

- `curl -b` / `--cookie` 现在会被解析成请求头里的 `Cookie` 字段，能直接在前端请求头面板中看到。
- 后端 HTTP 发送链路补齐了 Cookie 头注入逻辑，避免粘贴导入后请求和原始 `curl` 行为不一致。
- 增加了 Cookie 头相关的后端测试，降低后续改动把这条链路带坏的风险。

## 细节优化

- 响应头展示表格优化了左右列布局，长 value 不会再把 header key 挤没。
- 粘贴 `curl` 后仍然保持 URL 和查询参数表之间的同步逻辑，编辑体验更连贯。
