首个可用版本发布，带来一个面向桌面的 API 调试工作台。

## 核心能力

- 项目化工作区：基于项目、集合、请求三级结构组织接口，适合中小型 API 集合的持续整理与迭代。
- 多标签请求编辑：支持同时打开多个请求，在标签页之间快速切换、关闭、重排，并保留当前工作上下文。
- 请求构建器：支持常见 HTTP 方法，支持编辑 URL、Query、Headers、认证信息和请求体。
- 响应查看器：展示状态码、耗时、响应头、响应体和体积信息，对 JSON 响应提供格式化与高亮显示。

## 请求编辑能力

- Query 参数表与 URL 双向同步，修改任一侧都会自动更新另一侧。
- Header、Query、变量项支持启用/禁用，便于快速切换调试组合。
- 认证方式支持 `None`、`Basic`、`Bearer`、`API Key`。
- 请求体支持 `none`、`raw`、`json`，并预留了 `form-data`、`x-www-form-urlencoded`、`binary` 结构。
- Monaco 编辑器接入，适合编辑 JSON、脚本和较长文本内容。

## 工作区与组织能力

- 项目、集合、请求支持创建、编辑、删除。
- 集合与请求支持树形展示与拖拽排序，便于快速整理接口结构。
- 支持最近打开项目列表，方便在多个工作区之间切换。
- 启动时自动恢复上次打开项目与已打开标签页，减少重复进入成本。

## 环境与脚本

- 支持环境变量管理，可为项目维护多个环境。
- 支持设置当前激活环境，便于在不同后端地址之间切换。
- 支持 Pre-request Script 与 Post-request Script。
- 脚本环境提供 `fuck.config`、`fuck.response`、`fuck.env`、`fuck.vars` 等能力，可在请求前改写配置、在请求后读取和处理响应。
- Post-request Script 支持 `response.json(path)` 形式的 JSONPath 风格读取。

## 存储与桌面体验

- 采用基于文件夹与 JSON 的工作区存储模型，内容可读、可迁移、对 Git 友好。
- 基于 Tauri v2 构建，提供原生桌面窗口体验。
- 针对 macOS 做了窗口交通灯和拖拽区域适配。
- 支持自动更新通道，后续版本可通过 GitHub Releases 分发更新。

## 已知说明

- 当前请求体的实际发送能力以 `none`、`raw`、`json` 为主。
- macOS 构建当前使用 ad-hoc signing，首次打开时可能仍需手动授权。
