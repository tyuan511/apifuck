本次更新重点补齐了 WebSocket 调试能力与 HTTP 请求体编辑能力，让请求调试、消息收发和代码导出都更接近日常使用场景。同时也顺手优化了一些工作台里的弹窗和输入交互，让新建请求、发送消息这些高频动作更顺手。

## WebSocket 请求能力增强

- 项目树中的请求现在支持 WebSocket 类型，可直接创建并保存 `ws://` / `wss://` 请求。
- WebSocket 请求拥有独立的连接、断开和发送消息能力，并在同一页签中展示消息时间线。
- 时间线展示文案改得更直观，使用“请求 / 响应 / 系统 / 错误”代替底层的 `outbound / inbound` 等术语。
- 发送消息后会自动清空输入框，新的消息进入后时间线也会自动滚动到底部。
- WebSocket 编辑面板进一步收敛为更轻量的模型，不再展示请求体、请求脚本、响应脚本等不适用内容。
- 消息输入区改成顶部单行输入框，支持直接按回车发送，交互更接近即时调试面板。

## HTTP Body 能力补齐

- HTTP 请求体模式现在补全支持 `form-data`、`x-www-form-urlencoded` 和二进制文件上传。
- `form-data` 每一行支持切换为文本字段或文件字段，文件字段可直接从本地选择文件。
- 二进制请求体支持直接选择单个本地文件作为请求内容发送。
- 后端真正打通了上述 Body 模式，不再只是前端可编辑、发送时却报“不支持”的状态。

## 请求导出增强

- 当 HTTP 请求使用 `form-data` 文件字段时，导出代码现在会正确生成对应的文件上传写法。
- `cURL` 导出会使用 `--form 'field=@/path/to/file'` 形式表示文件上传。
- `JavaScript`、`Python`、`Go` 导出也同步支持文件型 `form-data`，生成更贴近真实可运行代码的示例。
- 导出逻辑继续保留环境变量解析与继承配置合并，文件型字段也能正确带入导出结果。

## 细节与交互优化

- WebSocket URL 校验与连接流程进一步修正，减少 `ws://` / `wss://` 地址被误判或握手失败的问题。
- 新建请求弹窗的请求类型选择器宽度优化，占满整行显示更自然。
- 请求体类型下拉宽度加宽，像 `x-www-form-urlencoded` 这类长名称可完整显示。
- 新建请求弹窗支持按回车直接提交，创建流程更流畅。
