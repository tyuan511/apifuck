本次更新重点重做了项目与目录层级的默认请求配置编辑体验，把原本零散、受弹窗限制的设置能力整合进工作台主界面，并补齐了一批实际使用中暴露出来的编辑器与交互问题。

## 项目与目录设置重构

- 项目和目录的请求头、认证、预请求脚本、后置脚本设置已移出弹窗，改为在右侧以独立标签页编辑。
- 点击目录节点时，会在右侧打开目录设置标签页；点击项目编辑时，会打开项目设置标签页。
- 目录树现在只有点击箭头才会展开或收起，点击目录本身会直接打开对应设置，不再误触折叠。
- 项目下拉菜单补充了更清晰的悬停操作，支持直接编辑和删除项目。

## 默认请求配置能力增强

- 新增项目级与目录级默认请求头编辑能力，可为上层作用域维护统一请求头。
- 新增项目级与目录级认证配置编辑能力，请求可继续继承这些上层认证设置。
- 请求级认证入口回归顶部下拉，项目与目录继续使用设置面板编辑，职责更清晰。
- 项目和目录设置中的请求头、认证相关输入已支持环境变量补全与编辑。
- 目录与项目默认请求头区域支持直接新增条目，补齐了与请求编辑页一致的操作方式。

## 编辑器与界面体验优化

- 修复预请求脚本和后置脚本 Monaco 编辑器高度异常的问题，编辑区现在可以正确占满可用空间。
- 调整 Monaco 编辑器主题与背景接入方式，编辑器底色会与应用当前主题背景保持一致。
- 当前激活行高亮改为使用主题的 `muted` 色，整体观感更统一。
- 响应加载中的状态指示器重新居中，减少视觉偏移。
- 项目头像改为更克制、稳定的随机配色，并统一了下拉内外的展示样式。

## 交互修复

- 修复点击左侧目录树中的请求时，无法正确切换到对应标签页的问题。
- 修复项目保存后仍错误提示“未保存”的问题。
- 修复默认项目无法删除的问题。
- 修复项目和目录认证设置只能启用继承、无法正常切换到其他认证方式的问题。
- 移除了多处冗余提示与展示项，包括项目 `slug`、请求页继承标签、目录继承说明等，界面更干净。
