本次更新聚焦在工作台细节打磨与桌面端体验优化，重点提升了响应查看、编辑器初始化、设置页信息组织和生产环境交互一致性。

## 响应查看体验升级

- 响应区域拆分为 `基本信息`、`响应头`、`响应体` 三个标签页，查看结构更清晰。
- 默认打开 `响应体` 标签页，发送请求后可以更快看到实际返回内容。
- 响应头支持独立查看，便于排查缓存、鉴权、跨域等问题。
- 基本信息面板集中展示状态码、耗时、大小、内容类型、响应头数量和请求状态。

## Monaco 编辑器体验优化

- Monaco 编辑器改为应用启动时预加载，不再等首次打开脚本或 JSON 编辑区时才初始化。
- 首次进入请求脚本、响应脚本和响应体视图时，编辑器出现更快、更稳定。
- 编辑器继续沿用应用主题背景与高亮样式，整体观感更统一。

## 设置页与关于页调整

- 设置页右侧改为固定标题区，滚动只发生在标题下方内容区域，浏览更稳定。
- 设置左侧菜单整体收紧，图标、间距和文案密度更紧凑。
- 主题模式文案从“亮色 / 暗色”调整为“浅色 / 深色”。
- 关于页移除了单独的应用名称与版本号信息卡片，版本号并入顶部主卡片。
- 关于页新增版本介绍区域，会直接展示当前版本对应的发布日志内容。

## 桌面交互与细节修复

- 默认右键菜单仅在生产构建中禁用，开发阶段保留，方便调试。
- 关于页应用图标样式更简洁，移除了多余包裹与边框装饰。
- 继续优化设置页视觉密度与信息层级，让界面更干净克制。
