本次更新带来了请求导出能力，并补强了请求 URL 的规范化与校验流程。现在你可以直接把当前接口复制成多种语言示例代码，同时在保存和发送请求时获得更稳妥的 URL 处理体验。

## 请求导出能力上线

- 请求右键菜单新增“复制为...”能力，可将当前接口一键导出为可直接使用的请求代码。
- 支持导出为 `cURL`、`JavaScript`、`Python`、`Go` 四种格式，便于快速分享给前后端同学或粘贴到脚本中验证。
- 导出时会自动合并项目级、目录级和请求级配置，尽量还原真实发送请求时的完整参数。
- 当前激活环境中的变量会在导出阶段被解析，生成的代码更接近最终实际请求内容。
- 认证信息会一并带入导出结果，覆盖 `Basic`、`Bearer`、`API Key` 等常见场景。
- 常见 Body 类型也已纳入导出支持，包括 `JSON`、原始文本、`form-data`、`x-www-form-urlencoded` 和二进制文件。

## URL 输入体验优化

- 请求 URL 在保存前会自动做规范化处理，减少因为多余空格或缺少协议头导致的保存异常。
- 针对像 `api.example.com`、`localhost:3000`、IPv4、IPv6 这类常见输入，现在会自动补全为可发送的 `https://` 地址。
- 对于 `/path`、`?query=1` 这类相对地址场景，依然保留支持，便于配合项目或环境 Base URL 使用。
- 发送请求前会先校验 URL 合法性，格式不正确时会直接给出提示，避免请求发出后才发现问题。
- URL 与 Query 参数之间的同步逻辑也同步补强，减少编辑过程中地址与参数状态不一致的情况。

## 细节改进

- 导出代码弹窗内置多语言切换和代码复制操作，查看与复制流程更顺手。
- 导出的请求代码会尽量补齐必要的请求头，比如 `Content-Type`，降低复制后还需要手工修补的概率。
- 围绕请求面板和工作台状态同步补做了一轮整理，为后续继续扩展环境与请求配置能力打下基础。
