本次更新重点补齐了默认 Base URL 配置、`text/event-stream` 流式响应展示，以及一批围绕请求树、加载状态和响应面板的交互细节修复。

## 流式响应体验升级

- 响应面板现已支持 `text/event-stream`，请求过程中可以持续接收并展示流式返回内容。
- SSE 响应会按 `data:` 行拆分为独立列表项展示，更适合查看大模型输出、日志流和增量事件。
- 响应区域会正确识别 `事件流` 视图类型，并在基本信息中显示对应状态。
- 针对 SSE 请求补充了更稳妥的传输处理，减少因为压缩、协议协商或客户端缓冲导致的延迟展示问题。

## Base URL 继承能力增强

- 项目设置新增 `项目 Base URL`，可以为整个工作区提供默认请求基地址。
- 目录设置新增 `目录 Base URL`，便于按模块或服务划分不同的请求前缀。
- 请求发送时会按“项目 -> 目录 -> 请求”链路解析 Base URL，并继续支持环境变量插值。
- 相关配置已接入存储层归一化处理，保存和读取行为更稳定。

## 响应与工作台细节优化

- 响应面板中的加载状态、事件流展示和滚动行为做了多轮调整，视觉更稳定。
- 所有主要 loading 指示器统一为一致的圆环样式，减少不同区域加载反馈不一致的问题。
- 请求详情加载、响应加载、Toast loading、检查更新等入口的 loading 样式已统一。
- 事件流列表文本换行、悬浮反馈和横向溢出表现做了优化，阅读更顺手。

## 交互修复

- 修复目录树行内出现 `<button>` 嵌套 `<button>` 导致的 hydration 报错。
- 目录与请求树行补齐键盘可访问性，支持使用 `Enter` / `Space` 触发行操作。
- 目录菜单移除了重复的“编辑目录”入口，避免与点击目录直接打开设置形成重复操作。
- 修复流式响应前端消息字段映射问题，避免因为字段命名不一致导致的实时更新失效。
