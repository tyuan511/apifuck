## Design Context

### Users
This product is a desktop API workbench for frontend engineers, backend engineers, QA, and API designers. It is used during active development and verification work where users need to inspect request structure, edit parameters quickly, send requests repeatedly, validate responses, and create or maintain mock definitions. The core job to be done is to make API design, debugging,联调, verification, and mock workflow feel fast, local, and dependable rather than heavy or process-driven.

### Brand Personality
The brand personality is: geeky, sharp, efficient.

The interface should feel technically confident, precise, and built for people who spend long uninterrupted sessions inside a tool. It should emphasize clarity, responsiveness, and control, with a strong sense of local ownership over data and workflow.

### Aesthetic Direction
The current product direction should continue toward a dense professional desktop workbench, closer to Bruno’s local-first feel than Postman’s heavier, more bloated visual impression.

Visual direction:
- both light mode and dark mode should be fully supported
- color usage should stay restrained, neutral, and purposeful
- avoid flashy, decorative, or overly saturated accents
- keep the interface lean, sharp, and structured
- prioritize scanability, editing efficiency, and low visual noise over visual spectacle

Anti-direction:
- do not feel bloated, bulky, or enterprise-dashboard-heavy
- do not use overly colorful or attention-seeking styling
- do not drift into marketing-site gradients, soft toy-like UI, or decorative card overload

### Design Principles
1. Build for real work: every screen should optimize active API editing, debugging, validation, and mock authoring rather than presentation.
2. Stay sharp and lean: prefer tight structure, crisp hierarchy, and efficient density over roomy but wasteful layouts.
3. Respect local-tool trust: the product should feel dependable, controllable, and native to technical workflows.
4. Use color with discipline: reserve emphasis for meaning and state, not decoration.
5. Design for long sessions: reduce noise, keep information scannable, and make repeated interactions feel fast and frictionless.
